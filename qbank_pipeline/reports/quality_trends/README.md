# Quality Trend Reports

Generated by:

```bash
cd app
npm run report:quality-trends
```

Outputs:
- `latest.json` / `latest.md` (rolling snapshot)
- timestamped snapshots (`YYYY-MM-DD_HHMMSS.json|md`)

Purpose:
- Monitor quality drift over time beyond binary pass/fail gates.
- Track whyWrong specificity, rationale quality, distractor plausibility, answer-key distribution, and review/submission pipeline trends.

Notes:
- This is currently an **observe-mode telemetry layer** (non-blocking).
- Existing CI quality gates remain the blocking controls.
